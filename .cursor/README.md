# Cursor Rules Boilerplate

This `.cursor/` folder contains a complete, self-contained Cursor Rules Boilerplate system that can be copied to new projects.

## Quick Start

1. **Copy this `.cursor/` folder** to your new project root
2. **Run bootstrap** in your new project:
   ```
   /bootstrap
   ```
   This will detect your project structure and generate foundational rules.

3. **Restart Cursor** to load the new rules

4. **Generate category rules**:
   ```
   /generate-category-rules
   ```
   This generates backend, frontend, and workflow rules with full project context.

```

│   ├── generate-feature.md       # Generate feature with rules
│   └── generate-debug-rule.md     # Generate debugging rules

│   ├── meta/           # Meta rules (rule system itself)
│   ├── general/        # Cross-cutting rules
│   ├── backend/        # Backend-specific rules
│   ├── frontend/       # Frontend-specific rules
│   └── workflow/       # Workflow/process rules
└── templates/          # Templates for rule/command generation
    ├── rules/          # Rule templates
    ├── commands/       # Command templates
    ├── debugging/      # Debugging rule templates
    ├── worktrees/      # Worktree configuration templates
    └── mcp/            # MCP configuration templates
├── worktrees.json      # Worktree configuration (auto-generated by bootstrap)
├── mcp.json            # MCP configuration (auto-generated by discover-mcp-servers)
├── README.md           # This file

└── MCP.md              # Complete MCP documentation
```

## Available Commands

- `/bootstrap` - Initialize the system (Phase 1) - **Auto-generates worktree config and offers MCP discovery**
- `/generate-category-rules` - Generate category rules (Phase 2)
- `/generate-rule` - Generate a single rule from templates
- `/generate-command` - Generate a new Cursor command
- `/generate-feature` - Generate a feature with automatic rule creation
- `/generate-debug-rule` - Generate debugging rules with MCP integration
- `/generate-worktree` - Generate or update worktree configuration
- `/discover-mcp-servers` - Discover and configure MCP servers (web-first priority)
- `/generate-mcp-config` - Generate or update MCP configuration

## Templates

All templates are self-contained in `.cursor/templates/`:

- **Rule Templates**: `templates/rules/*.mdc.template`
- **Command Templates**: `templates/commands/*.md`
- **Debugging Templates**: `templates/debugging/*.mdc.template`
- **Worktree Templates**: `templates/worktrees/*.json.template`

## Worktree Support (Parallel Agents)

The boilerplate includes automatic worktree configuration for Cursor's parallel agent feature:

- **Auto-generated during bootstrap**: `.cursor/worktrees.json` is created automatically
- **Project-aware**: Detects package manager, build tools, and database migrations
- **OS-specific**: Separate commands for Unix (macOS/Linux) and Windows
- **Customizable**: Use `/generate-worktree` to update configuration

See [WORKTREES.md](WORKTREES.md) for complete documentation.

## MCP Support (Model Context Protocol)

The boilerplate includes comprehensive MCP support with web development focus:

- **Auto-discovery during bootstrap**: Offers MCP server discovery based on project type
- **Web-first priority**: Prioritizes Linear, Figma, and Browser Tools for web projects
- **Mobile support**: Flutter and mobile development MCP servers
- **Project-aware**: Suggests relevant servers based on detected tools
- **Intelligent discovery**: Uses web search to find and suggest useful MCP servers

**Essential Web Development MCP Servers:**
- **Linear** - Project management integration
- **Figma** - Design system access
- **Browser Tools** - Console and network monitoring

See [MCP.md](MCP.md) for complete documentation.

## Meta Rules

The system includes three essential meta rules:

1. **rule-generator.mdc** - Validates rule structure and format
2. **rule-generation-guide.mdc** - Provides templates and checklists
3. **rule-improvement.mdc** - Self-improvement system guidelines

## Usage in New Projects

1. Copy this entire `.cursor/` folder to your project root
2. Run `/bootstrap` to detect project structure
3. Restart Cursor
4. Run `/generate-category-rules` to create project-specific rules
5. Use `/generate-rule` to create additional rules as needed
